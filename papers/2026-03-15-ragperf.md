# RAGPerf: An End-to-End Benchmarking Framework for Retrieval-Augmented Generation Systems

## Authors
Shaobo Li, Yirui Zhou, Yuan Xu, Kevin Chen, Daniel Waddington

## Abstract
We present the design and implementation of a RAG-based AI system benchmarking (**RAGPerf**) framework for characterizing the system behaviors of RAG pipelines.

To facilitate detailed profiling and fine-grained performance analysis, RAGPerf decouples the RAG workflow into several modular components:
- **Embedding**
- **Indexing**
- **Retrieval**
- **Reranking**
- **Generation**

RAGPerf offers the flexibility for users to configure the core parameters of each component and examine their impact on the end-to-end query performance and quality.

### Key Features
- **Workload generator** to model real-world scenarios by supporting diverse datasets (text, PDF, code, and audio)
- Different retrieval and update ratios, and query distributions
- Support for different embedding models
- Major vector databases: LanceDB, Milvus, Qdrant, Chroma, and Elasticsearch
- Different LLMs for content generation

### Metrics
- **Performance**: End-to-end query throughput, host/GPU memory footprint, CPU/GPU utilization
- **Accuracy**: Context recall, query accuracy, factual consistency

Evaluation shows that RAGPerf incurs negligible performance overhead.

## Why Notable
- **Comprehensive RAG benchmarking**: First end-to-end framework covering all RAG components
- **Production-ready**: Supports major vector databases and diverse data types
- **Open source**: Full codebase available on GitHub
