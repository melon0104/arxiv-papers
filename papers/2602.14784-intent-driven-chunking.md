# Intent-Driven Dynamic Chunking (IDC)

**arXiv ID:** 2602.14784  
**Published:** 2026-02-16  
**Categories:** cs.IR  
**Code:** https://github.com/unseen1980/IDC

## Authors
Christos Koutsiaris

## Summary
RAGやQAシステムにおける文書分割の新しいアプローチ。ユーザーの意図（クエリ予測）を活用して最適なチャンク境界を動的に決定。

## Key Contributions
- **意図駆動型セグメンテーション**: LLMでユーザーの意図を予測し、それに基づいてチャンク化
- **動的計画法**: 最適なチャンク境界を見つける大域最適化（貪欲法の落とし穴を回避）
- **Minimum Description Length基準**: 情報理論に基づくセグメンテーション品質の評価

## Technical Approach
1. LLMで対象文書に対する想定ユーザークエリを生成
2. 動的計画法で最適なチャンク境界を計算
3. 意図と文書構造の整合性を最大化

## Results
6種類の多様なQAデータセット（ニュース、Wikipedia、学術論文、技術文書）で評価：
- 5データセットでトップ1検索精度5-67%向上
- 6番目でも最良ベースラインと同等
- ベースラインより40-60%少ないチャンク数で93-100%の回答カバレッジ

## Significance
RAGシステムの検索性能を大幅に向上させる実用的な手法。特に長く異質な文書に対して効果が高い。固定長やコヒーレンスベースの従来手法の限界を克服。
