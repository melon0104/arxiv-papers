# LLM-Enhanced Log Anomaly Detection: A Comprehensive Benchmark of Large Language Models for Automated System Diagnostics

- **arXiv ID**: 2604.12218
- **カテゴリ**: cs.LG
- **投稿日**: 2026-04-14
- **URL**: https://arxiv.org/abs/2604.12218

## 選定理由
✅ **コード公開**: https://github.com/disha8611/llm-log-anomaly-benchmark

## 概要
ログ異常検知について、従来 parser+classifier、fine-tuned transformer、prompt-based LLM を横並びで比較した実務寄りベンチマーク。ラベルなし導入時のLLMの強さと限界を整理する。

## ポイント
1. HDFS/BGL/Thunderbird/Spirit の4データセットで包括比較。
2. fine-tuned transformer は高精度、LLM は **zero-shot即戦力** が強み。
3. accuracy だけでなく **cost / latency / label availability** まで見ている。

## 一言
運用監視にLLMを入れるかどうかの判断材料としてかなり使いやすい。
