# Adaptive Trie-Guided Decoding for Effective In-Document Query Auto-Completion

**arXiv:** https://arxiv.org/abs/2604.18257  
**カテゴリ:** cs.IR  
**投稿日:** 2026-04-20  
**コード:** https://github.com/rahcode7/DocQAC  
**データセット:** https://bit.ly/3IGEkbH

## 一言サマリー

GitHub公開・コード公開：ドキュメント内クエリ補完（DocQAC）に特化した適応型Trieガイドデコーディングを提案。T5/BARTでLLaMA-3やPhi-3を凌駕し実用性を実証。

## 概要

Webクエリ補完（WebQAC）と異なり、ドキュメント内検索特有の文脈（現在のドキュメント内容・クエリ履歴）を活用するDocQACを提案・研究。

**手法:**
- ユーザークエリプレフィックスを用いてLMを高品質な補完候補へソフトに誘導するTrie-ガイドデコーディング
- 調整可能なハイパーパラメータによる適応的ペナルティメカニズム（モデル確信度とTrie誘導のトレードオフ）
- RAGと軽量文脈信号（タイトル、キーフレーズ、サマリー）で文書コンテキストを効率統合

**結果:** ORCASから派生したDocQACベンチマークで、T5/BARTが既存の強力ベースラインと大型LLM（LLaMA-3、Phi-3）を凌駕。

## キーポイント

- 実ドキュメント検索環境に特化した新ベンチマーク構築（ORCAS由来）
- GitHub公開で再現性高
- エンコーダ・デコーダモデルの効率性と産業応用性を示す
